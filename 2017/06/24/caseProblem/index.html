git文件夹大小写问题

Jun 24, 2017 | Technology | 阅读

问题

在建立博客网站的过程中，遇到一个小问题，这里也记录一下吧。开始建立了两个博客分类，一个Emotion，一个technology。显然，大小写不一致让洁癖的我很不爽。于是我很乐观的将博文中categories中的标签均改成大写，重新部署到GitHub Pages上，打开博客主页，分类大小写一致，完美 😊。但当点击Technology分类时，却出现了404问题。

原因

看了下 github 项目目录，发现 categories 下两个文件夹仍为 Emotion、technology。这问题是由于 git 默认忽略文件名大小写，因此重新部署时文件夹名称并未修改。但是网站的超链接中使用的路径是大小写敏感的，于是出现了 404 错误。

解决方案

  • 将网站项目 clone 到本地，用 git rm <file/directory>删除大小写错误的文件或目录，重新 push 项目。
  • 使用 Hexo 重新生成并部署即可。
  • 为了避免再次发生类似的错误，可以进入到博客项目中 .deploy_git文件夹，修改 .git 下的 config 文件，将 ignorecase=true 改为 ignorecase=false，使 git 的配置变成大小写敏感。
